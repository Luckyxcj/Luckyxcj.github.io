## 1. 概述：为什么选型是第一道坎

MCU 选型决定了项目的硬件架构、软件生态、开发周期和成本天花板。选错一次，最轻也是重新设计原理图；最重的后果是量产阶段才发现 Flash 不够、外设不足、供货中断。

嵌入式项目的 MCU 选型需要在 **性能、功耗、成本、生态、供货** 五个维度之间做权衡。没有"最好的 MCU"，只有"最合适的 MCU"。

```
选型决策的五维空间：

        性能 (DMIPS/MHz, 主频)
           ↑
          /|\
         / | \
        /  |  \
       /   |   \
成本 ←-------→ 生态 (IDE, HAL, 社区)
       \   |   /
        \  |  /
         \ | /
          \|/
           ↓
        功耗 + 供货
```

## 2. MCU 核心参数体系

### 2.1 处理器内核

| 内核 | 架构 | DMIPS/MHz | 典型主频 | 代表系列 |
|------|------|-----------|----------|---------|
| Cortex-M0 | ARMv6-M | 0.87 | 48 MHz | STM32F0/G0, NXP LPC800 |
| Cortex-M0+ | ARMv6-M | 0.95 | 48 MHz | STM32L0/G0, Microchip SAM D |
| Cortex-M3 | ARMv7-M | 1.25 | 72-120 MHz | STM32F1/F2, NXP LPC1700 |
| Cortex-M4 | ARMv7E-M | 1.25 + DSP | 84-180 MHz | STM32F3/F4/G4, NXP Kinetis |
| Cortex-M7 | ARMv7E-M | 2.14 + DSP | 216-600 MHz | STM32F7/H7, NXP i.MX RT |
| Cortex-M33 | ARMv8-M | 1.50 + DSP | 96-160 MHz | STM32H5/L5, NXP LPC55xx |
| Cortex-M85 | ARMv8.1-M | 3.13 + Helium | 400-800 MHz | 最新旗舰 MCU |

### 2.2 存储配置

| 参数 | 低端 | 中端 | 高端 |
|------|------|------|------|
| Flash | 16-64 KB | 128-512 KB | 1-2 MB |
| SRAM | 4-16 KB | 32-256 KB | 512 KB-1 MB |
| EEPROM 模拟 | 需要软件模拟 | 部分型号内置 | 通常不需要 |
| 外部存储接口 | 无 | QSPI | FMC/SDRAM/QSPI/OSPI |

::: warning 选型陷阱
**Flash 容量评估时要留 30%-50% 余量**。需求变更、OTA 双区、Bug 修复都会消耗 Flash。实际项目中，固件体积通常会比初期估计大 1.5-2 倍。
:::

### 2.3 外设资源

外设是 MCU 选型的核心差异化因素。只凭主频和 Flash 大小选型，很容易遗漏关键外设需求。

| 外设类型 | 具体模块 | 典型用量 | 注意点 |
|----------|---------|---------|--------|
| 定时器 | 通用/高级/基本定时器 | 3-8 个 | 高级定时器用于电机控制 |
| ADC | 逐次逼近型 | 1-3 个, 8-16 通道 | SAR 分辨率 12-16 bit |
| DAC | 电压输出 | 1-2 通道 | 部分低价型号无 DAC |
| 通信 | USART/UART | 3-6 路 | 调试串口预留 1 路 |
| 通信 | I2C | 1-3 路 | 注意多主模式需求 |
| 通信 | SPI | 1-4 路 | SPI Flash 屏需高速 |
| 通信 | CAN/CAN-FD | 1-3 路 | 汽车/工业必须 |
| 通信 | USB | Device/Host/OTG | 区分 FS 和 HS |
| 通信 | Ethernet | MAC+PHY 或内置 PHY | MII/RMII 接口 |

### 2.4 封装与引脚

| 封装 | 引脚数 | 适用阶段 | 焊接难度 |
|------|--------|---------|---------|
| TSSOP-20 | 20 | 小空间 | 手工容易 |
| LQFP-48 | 48 | 原型/小批量 | 手工可焊 |
| LQFP-64 | 64 | 量产主流 | 手工可焊(需技巧) |
| LQFP-100 | 100 | 功能复杂 | 回流焊 |
| LQFP-144 | 144 | 高 IO 需求 | 回流焊 |
| BGA | 64-216+ | 空间极度受限 | 必须 X-ray 检测 |
| QFN | 32-64 | 体积敏感 | 底部焊盘需热风枪 |

## 3. 主流厂商与生态对比

### 3.1 生态系统评价维度

一个好的 MCU 生态包含：IDE 体验 → HAL/LL 库质量 → 社区活跃度 → 开发板丰富度 → FAE 支持 → 供货稳定性。

### 3.2 厂商对比总览

| 维度 | ST | NXP | TI | Microchip | GigaDevice | Espressif |
|------|-----|------|-----|-----------|------------|-----------|
| **核心产品线** | STM32 全系列 | Kinetis, LPC, i.MX RT | MSP430, TivaC, CC | PIC, AVR, SAM | GD32 (STM32替代) | ESP32 系列 |
| **IDE** | CubeIDE (免费) | MCUXpresso (免费) | CCS (免费) | MPLAB X (免费) | 兼容 Keil/IAR | ESP-IDF (开源) |
| **HAL库成熟度** | ★★★★★ | ★★★★ | ★★★ | ★★★ | ★★★★ | ★★★★ |
| **中文社区** | 最强 | 一般 | 一般 | 一般 | 增长快 | 强(IoT领域) |
| **开发板生态** | Nucleo/Discovery 极丰富 | FRDM 系列 | LaunchPad | Curiosity | 各类兼容板 | DevKit 系列 |
| **供货稳定性** | 2023年后改善 | 好 | 好 | 好 | 好(国产替代) | 好 |
| **性价比** | 中高 | 中 | 中 | 中低 | 高 | 极高(IoT) |

### 3.3 国产替代选择

| 原厂型号 | 国产替代 | 兼容度 | 注意事项 |
|----------|---------|--------|---------|
| STM32F103 | GD32F103 / APM32F103 / AT32F403 | 引脚兼容 | 需要核对 HSI 精度、Flash 等待周期差异 |
| STM32F407 | GD32F407 / AT32F407 | 高 | Bootloader 引脚可能不同 |
| STM32F030 | GD32F130 / AT32F415 | 中 | 外设寄存器不完全兼容 |
| STM32H750 | GD32H7 | 中 | 时钟系统和 Cache 有差异 |

::: tip 国产替代选型建议
先在目标平台上跑完整的 regression test（尤其是 ADC 精度、Flash 擦写时间、低功耗唤醒时间），不要仅凭引脚兼容就切换。
:::

## 4. 应用场景驱动的选型策略

### 4.1 消费电子（小家电、穿戴设备）

- **核心诉求**：低成本、低功耗、小封装
- **典型选型**：STM32F0/G0, MSP430, ESP32-C3, GD32F130
- **关键外设**：I2C（传感器）、UART（调试）、GPIO（按键/LED）

### 4.2 工业控制（PLC、变频器、仪表）

- **核心诉求**：高可靠性、宽温 -40~+85°C、CAN/RS-485、长供货周期
- **典型选型**：STM32F4/G4, NXP Kinetis K, TI TivaC, GD32F450
- **关键外设**：CAN-FD, UART×4, ADC 12-bit+, PWM 高精度, Ethernet

### 4.3 汽车电子（车身控制、传感器节点）

- **核心诉求**：AEC-Q100 认证、CAN/LIN、功能安全(ISO 26262)
- **典型选型**：STM32G0/A0 (成本敏感), STM32F3 (数字电源), S32K1 (NXP)
- **关键外设**：CAN-FD, LIN, ADC 多通道

### 4.4 IoT 终端（传感器节点、网关）

- **核心诉求**：无线连接(BLE/WiFi)、低功耗、OTA
- **典型选型**：ESP32, nRF52840, STM32WB, CC2652
- **关键外设**：BLE 5.0/WiFi, SPI Flash, ULP 协处理器

### 4.5 电机控制（FOC、BLDC、伺服）

- **核心诉求**：高分辨率 PWM (带死区)、高速 ADC 同步采样、FPU/DSP
- **典型选型**：STM32G4 (Cordic+Filter), STM32F3, TMS320F280xx
- **关键外设**：高级定时器 ×2, ADC 同步注入组, 比较器, 运放内置

## 6. 选型实战流程与决策矩阵

### 6.1 选型流程

```
需求分析 → 功耗预算 → 外设映射 → Flash/SRAM 估算 → 封装筛选 → 供货确认 → 样品测试 → 定型
```

**步骤详解**：

1. **需求分析**：列出所有外设及其数量（X 路 UART、Y 路 SPI、Z 路 ADC 通道...）
2. **功耗预算**：如果是电池供电，计算平均电流，确定是否需要低功耗系列
3. **外设映射**：确认目标 MCU 有足够的 DMA 通道、映射不冲突的引脚（参考 CubeMX）
4. **Flash/SRAM 估算**：编译一个空项目 + 估算库代码 + 应用代码 + 30% 余量
5. **封装筛选**：考虑 PCB 面积、层数、手工焊接/量产工艺
6. **供货确认**：在供货平台上查实时库存和交期（LCSC、Mouser、DigiKey）

### 6.2 决策矩阵模板

| 评估项 | 权重 | 候选A(STM32F407) | 候选B(GD32F450) | 候选C(ESP32-S3) |
|--------|------|------|------|------|
| 主频需求 | 15% | 168 (3分) | 200 (3分) | 240 (3分) |
| Flash/SRAM | 15% | 1MB/192KB (3分) | 2MB/256KB (4分) | 外挂 (2分) |
| CAN 接口 | 20% | 2路 (5分) | 2路 (5分) | 无 (0分) |
| 功耗 | 10% | 中 (3分) | 中 (3分) | 高 (1分) |
| 供货稳定性 | 15% | 好 (4分) | 很好 (5分) | 好 (4分) |
| 单价 | 15% | ¥25 (3分) | ¥18 (4分) | ¥20 (4分) |
| 开发资源(文档/论坛) | 10% | 极丰富 (5分) | 多 (4分) | 丰富 (4分) |
| **加权总分** | **100%** | **3.60** | **4.00** | **2.20** |

::: tip 加权评分说明
权重的赋值非常关键。工业场景下 CAN 接口可能是硬性需求（没有就一票否决），嵌入式消费场景下成本和功耗权重更高。不要机械套用，根据实际项目调整。
:::

## 7. 常见选型误区与排查

| # | 误区 | 后果 | 正确做法 |
|---|------|------|---------|
| 1 | 只看主频选型 | 遗漏关键外设，后期加外部芯片 | 先做外设需求映射，再对比 MCU |
| 2 | 完全不考虑封装 | 手工焊不了 BGA，或 PCB 面积不够 | 选型时就确认封装工艺可行性 |
| 3 | 只选"最新"系列 | 文档不全、社区支持少 | 量产项目优先选有 2 年以上历史的主力系列 |
| 4 | 忽略 Flash 扩展性 | 功能增加后 Flash 溢出 | 初始分配 Flash 余量 ≥30% |
| 5 | 不检查 DMA 通道冲突 | 某些外设无法同时使用 DMA | 用 CubeMX/ConfigTool 做引脚分配验证 |
| 6 | 只看芯片单价 | 忽略外围器件总成本 | 算 BOM 总成本(晶振/LDO/Flash/外部PHY等) |
| 7 | 不确认供货 | 量产阶段无货源 | 在多渠道查实时库存，避免 EOL 型号 |
| 8 | 忽略启动时间要求 | 对快速唤醒场景不满足 | 仔细对比低功耗唤醒时间（us 级要求） |

### 7.1 供货风险排查清单

- [ ] 在至少 2 个分销商平台上查实时库存（LCSC, Mouser, DigiKey）
- [ ] 确认型号生命周期状态（Active / NRND / EOL）
- [ ] 如果是新产品，确认是否已有量产出货记录
- [ ] 对于国产替代型号，至少焊接 5~10 片做全功能测试
- [ ] 确认芯片批次一致性和出厂固件版本
