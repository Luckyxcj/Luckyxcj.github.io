## 1. 协议对比总览

| 特性 | UART | I2C | SPI |
|------|------|-----|-----|
| 线数 | 2 (TX/RX) + GND | 2 (SCL/SDA) + GND | 4 (SCK/MOSI/MISO/CS) + GND |
| 通信方式 | 异步 | 同步 | 同步 |
| 最大速度 | ~6 Mbps (典型 115.2k-1M) | 100k / 400k / 1M / 3.4M | 数十 Mbps (通常 10-50 Mbps) |
| 拓扑 | 点对点 | 多主多从 (总线) | 一主多从 (星型) |
| 双工 | 全双工 | 半双工 | 全双工 |
| 从机寻址 | 无 (点对点) | 7/10 位地址 | 硬件片选 (CS) |
| 流控 | 可选 (RTS/CTS) | 内置 ACK/NACK | 无 |
| 从机能否主动通信 | 能 | 不能 (从机不能发起) | 不能 |
| 距离 | 长 (~15m RS-232) | 短 (~1m 板内) | 极短 (~10cm 板内) |

```
物理连接对比:

  UART (异步,2线)    I2C (同步,2线)       SPI (同步,4线)
  ┌────┐ ┌────┐     ┌────┐ ┌────┐     ┌────┐      ┌────┐
  │TX  ┼─→RX  │     │SCL ┼─┬─┤SCL │     │SCK ┼──────┤SCK │
  │RX  ┼←─TX  │     │SDA ┼─┼─┤SDA │     │MOSI┼──────→│MOSI│
  │GND ┼──GND │     │GND ┼─┘ ┤GND │     │MISO┼←──────│MISO│
  └────┘ └────┘     └────┘   └────┘     │CS  ┼──────→│CS  │
                   (可挂多个从机)        └────┘      └────┘
                                        │CS2 ┼──────→│CS  │
                                        └────┘      └────┘
                                       (每新增一个从机多加一根 CS)
```

## 2. 深入对比分析

### 2.1 UART

**优点**: 全双工、无时钟线、距离远、从机可主动发数据
**缺点**: 需约定波特率、点对点、无标准流控 (需额外硬件)

```
UART 帧格式 (如 8N1 = 8数据位, 无校验, 1停止位):

  ┌──┐  ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐  ┌──┐
  │  │  │0│ │1│ │2│ │3│ │4│ │5│ │6│ │7│ │P│  │  │
  │Id│  │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │  │Id│
  │le│  └┬┘ └┬┘ └┬┘ └┬┘ └┬┘ └┬┘ └┬┘ └┬┘ └┬┘  │le│
  └──┘   │   │   │   │   │   │   │   │   │   └──┘
   Start  D0  D1  D2  D3  D4  D5  D6  D7 Parity Stop
```

### 2.2 I2C

**优点**: 两线多设备、内置 ACK 确认、硬件地址自动筛选
**缺点**: 速度不如 SPI、从设备不能主动通知主机、总线死锁需恢复

```
I2C 时序 (SCL 低电平时 SDA 变化, SCL 高电平时 SDA 有效):

  SCL: ───┐   ┌───┐   ┌───┐   ┌──┐   ┌──┐   ┌──>  ┌───
          │   │   │   │   │   │  │   │  │   │  │   │
  ────────┘   └───┘   └───┘   └──┘   └──┘   └──┘   └──

  SDA: ────┐         ┌───┐           ┌───┐     ┌───
  ─────────└─────────┘   └───────────┘   └─────┘
           Start       D7 ... D0  ACK         Stop
```

### 2.3 SPI

**优点**: 全双工高速、无地址开销、简单推挽驱动、灵活时钟模式
**缺点**: 每个从机需独立 CS 线、无标准协议层 (无 ACK)、短距离

```
SPI 模式 0 (CPOL=0, CPHA=0) 时序:

  SCK:  ──┐   ┌───┐   ┌───┐   ┌───┐   ┌──
          │   │   │   │   │   │   │   │
  ────────┘   └───┘   └───┘   └───┘   └──

  MOSI: ───X───D7──X───D6──X───D5──X───D4──X──
  (CS↓ 开始传输, 每个 SCK 上升沿采样)
```

## 3. 电平转换与互联

### 3.1 常用电平标准

| 电平标准 | VIL max | VIH min | VOL max | VOH min |
|---------|---------|---------|---------|---------|
| 3.3V CMOS | 0.8V | 2.0V | 0.4V | 2.9V |
| 5V TTL | 0.8V | 2.0V | 0.4V | 2.4V |
| 5V CMOS | 1.5V | 3.5V | 0.4V | 4.4V |
| 1.8V CMOS | 0.6V | 1.17V | 0.35V | 1.45V |

### 3.2 3.3V ↔ 5V 转换方案

```c
// ===== 方案 1: 电阻分压 (单向, 低成本) =====
// 仅适用于 5V→3.3V, 低速 (<1 MHz)

//  5V TX ──[4.7k]─┬─[10k]── GND
//                  │
//                  └──→ 3.3V RX (约 3.3V)

// ===== 方案 2: 电平转换 IC =====
// 如 TXS0108E (8位双向自动方向)
//  A 侧 1.2-3.6V, B 侧 1.65-5.5V
//  速度: 推挽 60Mbps, 开漏 2Mbps

// ===== 方案 3: MOSFET 双向转换 =====
//  适用于 I2C (BSS138 + 2 个 10kΩ 上拉)

// 3.3V 侧                    5V 侧
//  ────┬────────────┐
//      │            │  D
//  3.3V 上拉       G ┼── BSS138
//      │            │  S
//  IO ─┴────────────┤
//                    └────┬──── 5V 上拉
//                         │
//                        IO

// ===== 方案 4: STM32 FT 引脚 (5V tolerant) =====
// F1/F4 系列多数 GPIO 默认为 FT (5V tolerant)
// 配置为开漏 + 外部上拉到 5V 即可直接驱动 5V 总线
```

### 3.3 I2C 长距离传输

```c
// I2C 板间长距离 (1m+):
// 1. 使用 I2C 缓冲器: PCA9615 (差分 I2C, 可达 3m)
// 2. 降低上拉电阻: 从 4.7k 降到 1k-2.2k
// 3. 降低速度: 从 400kHz 降到 100kHz 或 50kHz
// 4. 使用 I2C 转 CAN/RS-485: 长距离改用差分总线
```

## 4. 常见问题

| # | 问题 | 原因 | 解决方法 |
|---|------|------|---------|
| 1 | 3.3V MCU 直接连 5V 传感器，通信失败 | 3.3V 输出 VIH 勉强够但时序差 | 使用电平转换 |
| 2 | I2C 总线上电后死锁 (SDA 一直低) | 从机在上电/复位瞬间锁住总线 | 软件发送 9 个 SCL 脉冲 + STOP 恢复 |
| 3 | SPI 高速传输时数据错误 | 信号反射/串扰；CLK 与 Data 偏移 | 源端 22-33Ω 串联电阻；缩短走线 |
| 4 | UART 接收乱码 | 波特率偏差 >3% | 用外部晶振 (非内部 HSI)；校准时钟 |
| 5 | 多个 SPI 从机 CS 切换时数据错乱 | CS 切换间隙总线漂移 | CS 切换间加 1-2μs 延迟 |
| 6 | 3.3V MCU 的 I2C 与 5V 器件通信不稳定 | 逻辑电平不匹配 | 使用 MOSFET 双向转换或 TCA9548 等 I2C 转换器 |

## 5. 选型总结

```
需要长距离 (>1m)?
  └─ 是 → UART (RS-232/485)
是否需要多个设备 (>3 个) 共享 ≤3 条线?
  └─ 是 → I2C
需要高速全双工?
  └─ 是 → SPI
从机需要主动通知主机?
  └─ 是 → UART 或 SPI (需额外中断线)
最简单直接、调试方便?
  └─ UART (一个 USB-TTL 模块 + 串口助手即可)
```
