# CANopen 详细技术文档

## 1. CANopen 概述

CANopen 是基于 CAN 总线的**高层应用协议**，由 CiA (CAN in Automation) 组织制定和维护。它定义了应用层通信规范和设备配置文件，使得不同厂商的设备可以互联互通。

**协议栈层级关系：**

```
┌──────────────────────────────────────┐
│   CiA 4xx 设备子协议 (如 CiA 402)    │  ← 设备应用层
├──────────────────────────────────────┤
│   CANopen 应用层 (CiA 301)          │  ← NMT, PDO, SDO, 对象字典
├──────────────────────────────────────┤
│   CAN 数据链路层 (ISO 11898-1)      │  ← 帧格式、仲裁、错误检测
├──────────────────────────────────────┤
│   CAN 物理层 (ISO 11898-2/3)        │  ← 收发器、差分信号、终端电阻
└──────────────────────────────────────┘
```

**核心设计思想：**

- **对象字典 (Object Dictionary)**：所有设备参数统一编址，通过索引+子索引访问
- **服务数据对象 (SDO)**：点对点读写对象字典，类似"配置通道"
- **过程数据对象 (PDO)**：实时广播过程数据，类似"数据流通道"
- **网络管理 (NMT)**：统一管理所有节点的状态机
- **设备子协议**：为特定类型设备定义标准化对象字典 (如 CiA 401 I/O 模块、CiA 402 伺服驱动器)

## 2. 对象字典 (Object Dictionary)

对象字典是 CANopen 最核心的概念。每个设备的所有参数——从设备 ID 到应用数据——都存在对象字典中，通过 16 位索引 (Index) 和 8 位子索引 (Sub-index) 编址。

### 2.1 对象字典结构

```
索引范围            内容
──────────────────────────────────────────────
0x0000             保留
0x0001 – 0x001F    静态数据类型定义
0x0020 – 0x003F    复杂数据类型定义
0x0040 – 0x005F    制造商自定义数据类型
0x0060 – 0x007F    设备子协议静态数据类型
0x0080 – 0x009F    设备子协议复杂数据类型
0x00A0 – 0x0FFF    保留
0x1000 – 0x1FFF    通信对象区 (Communication Profile)
0x2000 – 0x5FFF    制造商自定义区
0x6000 – 0x9FFF    设备子协议区 (如 CiA 402 电机参数)
0xA000 – 0xAFFF    符合 IEC 61131-3 的网络变量
0xB000 – 0xBFFF    系统变量 (CiA 302)
0xC000 – 0xFFFF    保留
```

### 2.2 通信对象区 (0x1000 – 0x1FFF) 关键条目

| 索引 | 名称 | 说明 |
|------|------|------|
| 0x1000 | 设备类型 | 低 16 位为设备子协议号 (如 0x0191=CiA 401, 0x0192=CiA 402) |
| 0x1001 | 错误寄存器 | bit0=一般错误, bit1=电流, bit2=电压, bit3=温度…… |
| 0x1003 | 预定义错误域 | 最近 N 个紧急错误历史 |
| 0x1005 | COB-ID SYNC | 同步报文 COB-ID |
| 0x1006 | 通信周期 | 同步报文发送周期 (μs) |
| 0x1008 | 设备名 | 制造商自定义设备名称 (字符串) |
| 0x1009 | 硬件版本 | 硬件版本号 (字符串) |
| 0x100A | 软件版本 | 固件版本号 (字符串) |
| 0x100C | 监护时间 | Heartbeat 或 Node Guarding 周期 (ms) |
| 0x100D | 生命因子 | Node Guarding 生命因子 |
| 0x1010 | 存储参数 | 保存参数到非易失存储 |
| 0x1011 | 恢复默认参数 | 恢复出厂设置 |
| 0x1014 | COB-ID EMCY | 紧急报文 COB-ID |
| 0x1016 | 心跳消费者 | 监控目标节点的心跳周期和超时 |
| 0x1017 | 心跳生产者 | 本节点心跳发送间隔 (ms) |
| 0x1018 | 标识对象 | 子索引 1=Vendor-ID, 2=Product-Code, 3=Revision, 4=Serial |
| 0x1200 – 0x127F | SDO 服务器参数 | SDO 通道配置 |
| 0x1400 – 0x15FF | RPDO 通信参数 | 接收 PDO 的 COB-ID、传输类型 |
| 0x1600 – 0x17FF | RPDO 映射参数 | 接收 PDO 映射到哪些对象字典条目 |
| 0x1800 – 0x19FF | TPDO 通信参数 | 发送 PDO 的 COB-ID、传输类型、禁止时间 |
| 0x1A00 – 0x1BFF | TPDO 映射参数 | 发送 PDO 映射哪些对象字典条目 |
| 0x1F80 | NMT 启动行为 | 上电后是否自动进入 Operational 状态 |

## 3. 网络管理 (NMT)

### 3.1 NMT 状态机

CANopen 定义了一个标准状态机，每个从节点必须实现：

```
                         上电/硬件复位
                              │
                              ▼
                     ┌────────────────┐
              ┌────  │ Initialisation │
              │      └───────┬────────┘
              │              │ 初始化完成，自动发送 Boot-up
              │              ▼
              │      ┌────────────────┐
              │ ┌──→ │  Pre-Operational│ ←──────────────┐
              │ │    └───┬─────┬──────┘                │
              │ │        │     │                       │
              │ │        │     │ NMT: Start             │
              │ │  NMT:  │     │ Remote Node            │
              │ │  Enter │     ▼                       │
              │ │Pre-Oper│ ┌──────────────┐    NMT:    │
              │ │        │ │ Operational  │──→ Enter   │
              │ │        │ └──────┬───────┘  Pre-Oper  │
              │ │        │        │                     │
              │ │        │        │ NMT: Stop           │
              │ │        ▼        ▼                     │
              │ │      ┌────────────────┐              │
              │ └───── │    Stopped     │──────────────┘
              │        └────────────────┘
              │
              ▼
        ┌──────────┐
        │  Reset   │ (NMT: Reset Node / Reset Communication)
        └──────────┘
```

各状态说明：

| 状态 | 允许的操作 | 说明 |
|------|-----------|------|
| **Initialisation** | 无 | 上电自检、对象字典初始化。完成后自动发送 Boot-up 报文 |
| **Pre-Operational** | SDO, NMT, EMCY, SYNC, Heartbeat | PDO 被禁止，参数配置阶段 |
| **Operational** | 全部 | PDO 使能，正常数据交换阶段 |
| **Stopped** | NMT, Heartbeat (可选) | 仅响应 NMT 命令，停止所有通信 |

### 3.2 NMT 报文格式

NMT 报文使用 **COB-ID = 0x000** (最高优先级)，格式固定：

```
COB-ID  Byte0    Byte1
0x000   Node-ID  Command

Command 含义：
  0x01  Start Remote Node        → Operational
  0x02  Stop Remote Node         → Stopped
  0x80  Enter Pre-Operational    → Pre-Operational
  0x81  Reset Node               → 复位应用层
  0x82  Reset Communication      → 复位通信层
```

**Node-ID = 0x00 表示广播，所有节点响应。**

示例 — 将节点 3 切换到 Operational：

```
COB-ID: 0x000
Data:   0x03  0x01
       (节点3)(Start)
```

### 3.3 Boot-up 报文

节点初始化完成后自动发送一条 Boot-up 报文，使用 **COB-ID = 0x700 + Node-ID**，数据为 1 字节 0x00。

```
节点 1 的 Boot-up → COB-ID: 0x701, Data: [0x00]
节点 5 的 Boot-up → COB-ID: 0x705, Data: [0x00]
```

## 4. 服务数据对象 (SDO)

SDO 提供**点对点、客户端-服务器模式**的对象字典访问，用于参数配置。基于"请求-确认"模型，每次通信都有确认。

### 4.1 COB-ID 分配

| 方向 | COB-ID |
|------|--------|
| 客户端 → 服务器 (下载/读请求) | 0x600 + Node-ID |
| 服务器 → 客户端 (上传/写响应) | 0x580 + Node-ID |

### 4.2 SDO 报文格式

**SDO 读取 (Upload) — 客户端请求服务器发送数据：**

```
客户端 → 服务器:
Byte0: 0x40              (CCS=2: Upload Request, 无分段)
Byte1-2: Index (小端)    要读取的索引
Byte3: Sub-index        要读取的子索引
Byte4-7: 保留 (0x00)

服务器 → 客户端 (成功):
Byte0: 0x4x              (SCS=2: Upload Response, x=数据字节数-1)
Byte1-2: Index (小端)
Byte3: Sub-index
Byte4-7: 数据 (小端, 最多 4 字节)
```

**SDO 写入 (Download) — 客户端向服务器写入数据：**

```
客户端 → 服务器:
Byte0: 0x22              (SCS=1: Download Request, 无分段, 4 字节)
Byte1-2: Index (小端)
Byte3: Sub-index
Byte4-7: 数据 (小端)

服务器 → 客户端 (成功):
Byte0: 0x60              (SCS=3: Download Response)
Byte1-2: Index (小端)
Byte3: Sub-index
Byte4-7: 保留 (0x00)
```

### 4.3 SDO 报文示例

**示例 1：读取节点 2 的设备类型 (0x1000, 子索引 0x00)**

```
客户端请求 (COB-ID = 0x602):
  40 00 10 00 00 00 00 00
   │ └─Index:0x1000──┘
   └─Upload Request

服务器响应 (COB-ID = 0x582):
  43 00 10 00 92 01 00 00
   │ └─Index──┘ └─数据──┘
   └─Upload Response, 4 字节
   数据 0x0192 = CiA 402 设备
```

**示例 2：设置节点 2 的心跳周期为 500ms (0x1017, 子索引 0x00)**

```
客户端请求 (COB-ID = 0x602):
  2F 17 10 00 F4 01 00 00
   │ └─0x1017──┘ └数据──┘
   └─Download Request, 2 字节
  数据 0x01F4 = 500 (ms)

服务器响应 (COB-ID = 0x582):
  60 17 10 00 00 00 00 00
   └─Download Response 成功
```

## 5. 过程数据对象 (PDO)

PDO 用于**实时 I/O 数据传输**，优势是开销极小、无协议头、无需确认，适合循环同步数据（如电机位置、速度指令）。

### 5.1 PDO 特性对比

| 特性 | PDO | SDO |
|------|-----|-----|
| 通信模式 | 生产者-消费者 (广播) | 客户端-服务器 (点对点) |
| 确认机制 | 无 (可中断) | 有 (每次响应) |
| 最大数据 | 8 字节 | 无限制 (通过分段) |
| 使用时机 | 运行时循环数据 | 配置/诊断/参数读写 |
| 优先级 | 通常较低 COB-ID | 固定 COB-ID |

### 5.2 PDO COB-ID 默认分配

CANopen 预定义了 4 个 TPDO 和 4 个 RPDO：

| PDO | 功能 | 默认 COB-ID |
|-----|------|-------------|
| TPDO1 | 发送过程数据 1 | 0x180 + Node-ID |
| TPDO2 | 发送过程数据 2 | 0x280 + Node-ID |
| TPDO3 | 发送过程数据 3 | 0x380 + Node-ID |
| TPDO4 | 发送过程数据 4 | 0x480 + Node-ID |
| RPDO1 | 接收过程数据 1 | 0x200 + Node-ID |
| RPDO2 | 接收过程数据 2 | 0x300 + Node-ID |
| RPDO3 | 接收过程数据 3 | 0x400 + Node-ID |
| RPDO4 | 接收过程数据 4 | 0x500 + Node-ID |

**重要**：COB-ID 最低位 (bit31) 设为 1 表示该 PDO 无效 (禁用)。

### 5.3 PDO 传输类型

通过通信参数 (0x1400/0x1800) 的子索引 2 设置:

| 传输类型值 | 触发方式 | 说明 |
|-----------|---------|------|
| 0x00 | 同步、非周期 | 收到 SYNC 后，数据变化才发送 |
| 0x01 – 0xF0 | 同步、周期 | 收到 N 个 SYNC 后发送 (如 0x01=每个 SYNC 都发) |
| 0xF1 – 0xFB | 保留 | — |
| 0xFC | RTR 远程请求 | 收到远程帧后发送 (极少使用) |
| 0xFD | RTR 远程请求 | 收到远程帧后发送 (极少使用) |
| 0xFE – 0xFF | 异步 | 事件驱动 (数据变化即发送) |

**常用组合：**

```
同步周期模式 (推荐用于位置环/速度环):
  TPDO 传输类型 = 0x01 → 每个 SYNC 周期发送当前位置/速度/电流

异步模式 (推荐用于状态变化通知):
  TPDO 传输类型 = 0xFF → 状态字变化时立即发送
```

### 5.4 PDO 映射

PDO 映射决定了"这 8 个字节里装什么"。映射参数在：
- **RPDO**: 0x1600 – 0x17FF
- **TPDO**: 0x1A00 – 0x1BFF

**映射格式：** 每个映射条目为 4 字节：`[Index(16bit)][Sub-index(8bit)][Bit-Length(8bit)]`

**映射示例 — TPDO1 映射三个对象：**

```
TPDO1 映射参数 (0x1A00):
  子索引 0: 0x03           (映射了 3 个对象)
  子索引 1: 0x60410010     (Index=0x6041, Sub=0x00, 16bit → 状态字)
  子索引 2: 0x60640020     (Index=0x6064, Sub=0x00, 32bit → 实际位置)
  子索引 3: 0x606C0020     (Index=0x606C, Sub=0x00, 32bit → 实际速度)
                    └─ 总数据长度 = 2+4+4 = 10 字节
```

::: warning 注意
总映射数据长度不能超过 8 字节 (CAN 2.0 单帧数据上限)。映射超出 8 字节会导致 SDO 错误。
:::

### 5.5 PDO 配置流程 (SDO 方式)

以配置节点 2 的 TPDO1 为例（使其发送状态字+实际位置+实际速度）：

```
步骤 1: 禁止 TPDO1
  写 0x1800.01 = 0x80000182  (bit31=1, 禁用)
  → COB-ID: 0x602  Data: 23 00 18 01 82 01 00 80

步骤 2: 清空映射
  写 0x1A00.00 = 0x00 (映射条目数 = 0)

步骤 3: 写入映射条目
  写 0x1A00.01 = 0x60410010 (状态字, 16bit)
  写 0x1A00.02 = 0x60640020 (实际位置, 32bit)
  写 0x1A00.03 = 0x606C0020 (实际速度, 32bit)

步骤 4: 设置映射条目数
  写 0x1A00.00 = 0x03

步骤 5: 设置传输类型
  写 0x1800.02 = 0x01 (同步周期, 每个 SYNC 发送)

步骤 6: 恢复 COB-ID, 使能 TPDO1
  写 0x1800.01 = 0x40000182 (bit31=0, RTR 允许)
  → COB-ID: 0x602  Data: 23 00 18 01 82 01 00 40
```

## 6. 特殊通信对象

### 6.1 SYNC (同步报文)

- **COB-ID**: 0x080 (默认, 可通过 0x1005 修改)
- **数据长度**: 0 字节
- **功能**: 作为时间基准，触发同步 PDO 的发送
- **发送者**: SYNC 生产者 (通常为 NMT 主站)
- **周期**: 由对象 0x1006 定义 (μs)

```
典型用法 — 每个 SYNC 周期:
  1. 主站发送 SYNC (COB-ID 0x080, DLC=0)
  2. 所有从站收到 SYNC 后，立刻发送 TPDO
  3. 主站收到所有 TPDO 后，计算控制量
  4. 主站发送 RPDO (控制指令)
  5. 下一个 SYNC 到来，循环往复
```

### 6.2 EMCY (紧急报文)

- **COB-ID**: 0x080 + Node-ID
- **数据格式**: 8 字节 `[错误码 L][错误码 H][错误寄存器][厂商错误域 5 字节]`
- **功能**: 设备发生错误时主动上报

**错误码分类：**

| 错误码 | 含义 |
|--------|------|
| 0x1000 | 一般错误 |
| 0x2300 | 电流错误 (连续过流) |
| 0x3100 | 电压错误 (欠压/过压) |
| 0x4200 | 温度错误 (过温) |
| 0x5000 | 硬件错误 |
| 0x6100 | 软件内部错误 |
| 0x7300 | 位置传感器错误 |
| 0x8100 | 通信错误 |
| 0xFF00 | 厂商自定义 |

**示例 — 节点 3 发生过流报警：**

```
COB-ID: 0x083
Data:   00 23 00 01 00 00 00 00
         └────┘ └错误寄存器──┘
        0x2300
    (电流错误)  (一般错误)
```

### 6.3 Heartbeat (心跳)

- **COB-ID**: 0x700 + Node-ID
- **数据**: 1 字节，表示当前 NMT 状态 (0x04=Stopped, 0x05=Operational, 0x7F=Pre-Operational)
- **周期**: 由对象 0x1017 定义 (ms)

**Heartbeat 消费者机制 (节点监控)：**

某个节点可以在 0x1016 中配置监控列表，如果被监控节点在 `周期 × 因子` 时间内未发送心跳，则认为对方离线，触发内部错误。

```
示例 — 配置节点 1 监控节点 2 (期望周期 500ms, 超时因子 2):
  写 0x1016.01 = 0x000201F4
                └节点2┘└500ms─┘
  超时 = 500ms × 2 = 1000ms
  如果 1000ms 内未收到节点 2 心跳 → 触发 Heartbeat Event
```

### 6.4 TIME (时间戳)

- **COB-ID**: 0x100
- **数据**: 6 字节，绝对时间 (日-时-分-秒-毫秒)

## 7. CiA 402 电机驱动标准

CiA 402 (原 DSP 402) 是 CANopen 为**伺服驱动器、变频器、步进电机**定义的标准设备子协议，它规范了电机控制的对象字典和状态机。

### 7.1 控制状态机 (Power Drive System FSA)

这是 CiA 402 的核心——一个严格的、统一的控制状态机：

```
                                    ┌──────────────┐
                        上电  ──→    │  Not Ready   │
                                    │   to Switch  │
                                    │    On        │
                                    └──────┬───────┘
                                           │ 自动跳转
                                           ▼
                                    ┌──────────────┐
                                    │   Switch On  │
                                    │   Disabled   │
                                    └───┬─────┬────┘
                           Shutdown │   │     │ Quick Stop
                          (0x06)    │   │     │ (0x02)
                    ┌───────────────┘   │     └──────────────┐
                    ▼                   ▼                    ▼
             ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
             │   Ready to   │   │   Switched   │   │ Quick Stop   │
             │   Switch On  │←──│    On        │──→│   Active     │
             └──────┬───────┘   └──────┬───────┘   └──────────────┘
                    │   Switch On      │
                    │   (0x07)         │ Enable Operation (0x0F)
                    │                  ▼
                    │          ┌──────────────┐
                    └─────────→│   Operation  │
                               │   Enabled    │
                               └──────┬───────┘
                                      │ Disable Operation (0x07)
                                      ▼
                               ┌──────────────┐
                               │   Switched   │
                               │    On        │
                               └──────────────┘
```

### 7.2 关键对象字典 (CiA 402)

**控制与状态：**

| 索引 | 名称 | 类型 | 说明 |
|------|------|------|------|
| 0x6040 | 控制字 (Controlword) | U16 | 控制状态机跳转 |
| 0x6041 | 状态字 (Statusword) | U16 | 当前状态反馈 |

**控制模式：**

| 索引 | 名称 | 说明 |
|------|------|------|
| 0x6060 | 运行模式 (Modes of Operation) | 选择控制模式 |
| 0x6061 | 运行模式显示 (Modes of Operation Display) | 当前生效模式 |

**运行模式对应值：**

| 值 | 模式 | 说明 |
|----|------|------|
| 0 | — | 无模式切换 |
| 1 | 轮廓位置模式 (PP) | 按轨迹规划控制位置 |
| 3 | 轮廓速度模式 (PV) | 按轨迹规划控制速度 |
| 4 | 轮廓转矩模式 (PT) | 按轨迹规划控制转矩 |
| 6 | 回零模式 (HM) | 找原点/限位 |
| 7 | 插补位置模式 (IP) | 外部提供位置指令 |
| 8 | 周期同步位置模式 (CSP) | 每个周期同步位置指令 |
| 9 | 周期同步速度模式 (CSV) | 每个周期同步速度指令 |
| 10 | 周期同步转矩模式 (CST) | 每个周期同步转矩指令 |

## 8. 驱动电机实战：CSP 模式完整流程

下面通过 CiA 402 CSP (周期同步位置模式) 驱动一个伺服电机（节点 ID = 2），展示完整的报文序列。

### 8.1 系统架构

```
    CANopen 主站 (PLC/工控机/嵌入式)
           │
           │ CAN 总线
           │
    ┌──────┴──────┐
    │   伺服驱动器  │  ← Node-ID = 2
    │   (从站)     │
    └──────┬──────┘
           │
        ┌──┴──┐
        │ 电机 │
        └─────┘
```

### 8.2 完整操作流程

#### 阶段 1：上电与初始化

```
① 从站上电后自动完成初始化，发送 Boot-up:
  COB-ID: 0x702  Data: [00]
```

#### 阶段 2：配置运行参数 (通过 SDO)

```
② 设置运行模式为 CSP (模式 8):
  请求: COB-ID: 0x602  Data: 2F 60 60 00 08 00 00 00
        └─SDO Download 2byte, 写 0x6060.00 = 0x08
  响应: COB-ID: 0x582  Data: 60 60 60 00 00 00 00 00
        └─SDO Download Response, 成功

③ 配置 PDO 映射 — 设置 RPDO1 (接收目标位置):
  步骤 3.1: 禁止 RPDO1
    请求: COB-ID: 0x602  Data: 23 00 14 01 02 02 00 80
          写 0x1400.01 = 0x80000202 (bit31=1 禁用)

  步骤 3.2: 清空映射
    请求: COB-ID: 0x602  Data: 2F 00 16 00 00 00 00 00
          写 0x1600.00 = 0x00

  步骤 3.3: 映射控制字 (0x6040, 16bit)
    请求: COB-ID: 0x602  Data: 23 01 16 00 10 00 40 60
          写 0x1600.01 = 0x60400010

  步骤 3.4: 映射目标位置 (0x607A, 32bit)
    请求: COB-ID: 0x602  Data: 23 02 16 00 20 00 7A 60
          写 0x1600.02 = 0x607A0020

  步骤 3.5: 设置映射条目数
    请求: COB-ID: 0x602  Data: 2F 00 16 00 02 00 00 00
          写 0x1600.00 = 0x02

  步骤 3.6: 设置传输类型 = 同步 (0x01)
    请求: COB-ID: 0x602  Data: 2F 00 14 02 01 00 00 00
          写 0x1400.02 = 0x01

  步骤 3.7: 恢复 COB-ID, 使能 RPDO1
    请求: COB-ID: 0x602  Data: 23 00 14 01 02 02 00 40
          写 0x1400.01 = 0x40000202

④ 同样方法配置 TPDO1 映射 (状态字 + 实际位置 + 实际速度):
  TPDO1 COB-ID = 0x182
  映射条目:
    0x1A00.01 = 0x60410010  (状态字, 16bit)
    0x1A00.02 = 0x60640020  (实际位置, 32bit)
    0x1A00.03 = 0x606C0020  (实际速度, 32bit)
  传输类型 = 0x01 (每个 SYNC 发送)
```

#### 阶段 3：进入运行状态 (NMT + 控制字)

```
⑤ NMT 命令：节点 2 进入 Operational
  COB-ID: 0x000  Data: 02 01
                        └Start──┘

⑥ 控制字状态机跳转 — 通过 PDO 发送控制字

  步骤 1: Shutdown → Ready to Switch On
    RPDO1: COB-ID: 0x202  Data: 06 00 00 00 00 00 00 00
           控制字 = 0x0006

  步骤 2: Switch On → Switched On
    RPDO1: COB-ID: 0x202  Data: 07 00 00 00 00 00 00 00
           控制字 = 0x0007

  步骤 3: Enable Operation → Operation Enabled
    RPDO1: COB-ID: 0x202  Data: 0F 00 00 00 00 00 00 00
           控制字 = 0x000F

  每到某个状态时，检查 TPDO1 (COB-ID 0x182) 中的状态字:
    状态字 bit0-6 = 0x21 → Ready to Switch On
    状态字 bit0-6 = 0x23 → Switched On
    状态字 bit0-6 = 0x27 → Operation Enabled
```

#### 阶段 4：运行控制 (CSP 同步周期)

```
⑦ 主站周期性发送 SYNC:
  COB-ID: 0x080  Data: (空, DLC=0)

⑧ 每个 SYNC 周期:
  主站: 通过 RPDO1 发送 控制字 + 目标位置
  从站: 收到 SYNC 后通过 TPDO1 回复 状态字 + 实际位置 + 实际速度

  示例 — 目标位置 = 100000 counts (0x000186A0):
    RPDO1: COB-ID: 0x202  Data: 0F 00 A0 86 01 00 00 00
                               控制字=0x000F  目标位置=0x000186A0

  从站反馈 (TPDO1):
    COB-ID: 0x182  Data: 27 06 50 85 01 00 15 00
                            状态字=0x0627  位置=0x00018550  速度=0x0015
                            bit10=1        99664 counts      21 rpm
                            (Target Reached)
```

### 8.3 控制字命令速查表

| 跳转目标 | 控制字 bit | 命令值 | bit 说明 |
|---------|-----------|--------|---------|
| Shutdown | — | 0x0006 | bit0=0, bit1=1, bit2=1 |
| Switch On | — | 0x0007 | bit0=1, bit1=1, bit2=1 |
| Switch On + Enable Operation | — | 0x000F | bit0=1, bit1=1, bit2=1, bit3=1 |
| Disable Voltage | — | 0x0000 | 全部清零 (紧急停机) |
| Quick Stop | — | 0x0002 | bit1=1 |
| Disable Operation | — | 0x0007 | bit3 清零 |
| Enable Operation | — | 0x000F | bit3 置 1 |
| Fault Reset | 0x0080 | bit7=0→1 上升沿 | 故障确认 |

### 8.4 状态字解析

```
状态字 (0x6041) 位定义:
  bit0:  Ready to Switch On
  bit1:  Switched On
  bit2:  Operation Enabled
  bit3:  Fault
  bit4:  Voltage Enabled (主电已使能)
  bit5:  Quick Stop
  bit6:  Switch On Disabled
  bit7:  Warning
  bit8:  厂商自定义
  bit9:  Remote (0=本地控制, 1=总线控制)
  bit10: Target Reached (目标已到达)
  bit11: Internal Limit Active (内部限位有效)
  bit12-13: 运行模式相关
  bit14-15: 厂商自定义

常见状态字值与状态对应:
  0x0040 → Switch On Disabled (主电未通)
  0x0021 → Ready to Switch On
  0x0023 → Switched On
  0x0027 → Operation Enabled (正常运行)
  0x0008 → Fault (故障)
  0x000F → Fault (故障, 含警告)
```

## 9. 控制模式切换注意事项

### 9.1 模式切换前提条件

**模式切换只能在 Operation Enabled 状态下进行。**

尝试在其他状态切换模式会被驱动器忽略或报错。

### 9.2 切换流程

```
1. 确保当前状态 = Operation Enabled (状态字 bit0-6 = 0x27)
2. 通过 SDO 或 PDO 写入新的模式到 0x6060
3. 等待模式切换完成 (读取 0x6061 确认新模式已生效)
4. 按新模式的规则写入目标值
```

### 9.3 PP (轮廓位置) → CSP (周期同步位置) 切换示例

```
当前: PP 模式 (0x6060 = 0x01), 状态 Operation Enabled

步骤 1: 电机先停止轮廓运动
  控制字写入 0x010F (bit8=Halt, 停止当前运动)

步骤 2: 等待电机停止
  检查状态字 bit10 (Target Reached) = 1

步骤 3: 写入新运行模式
  SDO 写 0x6060.00 = 0x08 (CSP)

步骤 4: 验证模式生效
  SDO 读 0x6061.00, 确认返回 0x08

步骤 5: CSP 模式下，目标位置 (0x607A) 在每个 SYNC 周期生效
  RPDO 每个周期更新目标位置到 0x607A
```

### 9.4 各模式特性对比

| 特性 | PP | PV | CSP | CSV | CST | HM |
|------|----|----|-----|-----|-----|-----|
| 主站规划轨迹 | ✓ | ✓ | ✗ | ✗ | ✗ | — |
| 实时性要求 | 低 | 低 | 高 | 高 | 高 | — |
| 多轴同步 | 差 | 差 | 好 | 好 | — | — |
| PDO 数据 | 位置 | 速度 | 位置 | 速度 | 转矩 | 模式相关 |
| 控制字 bit4-6 | 有特殊用途 | 有特殊用途 | 不使用 | 不使用 | 不使用 | 有特殊用途 |

### 9.5 模式切换常见错误

| 错误现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 切换后电机不响应 | 未进入 Operation Enabled | 检查状态字, 按状态机步骤使能 |
| 切换后模式不生效 | 0x6060 写入失败 | 检查 SDO 响应, 确认无错误码 |
| 模式切换时驱动器报故障 | 不当前模式下执行了非法操作 | 先停止当前运动再切换 |
| CSP 模式下位置跟踪误差大 | SYNC 周期不匹配 | 检查 0x1006 (通信周期) 是否与主站一致 |
| 无法写 0x6060 | PDO/SDO 配置错误 | 确认节点在 Operational 状态, SDO 通道未占用 |

### 9.6 Quick Stop 行为

Quick Stop (控制字 0x0002) 在不同模式下的行为：

| 当前模式 | Quick Stop 行为 | 恢复方式 |
|---------|----------------|---------|
| PP | 立即停止 (可选减速) | 重新走 Shutdown → Switch On → Enable |
| CSP | 驱动器立即停止电机 | 重新走 Shutdown → Switch On → Enable |
| CSV | 以 Quick Stop 减速度减速到 0 | 重新走 Shutdown → Switch On → Enable |
| HM | 中断回零, 停止 | 重新启动回零 |

Quick Stop 结束后状态字 bit5 = 1 (Quick Stop Active)，此时只能通过控制字 0x0006 (Shutdown) 退出。

## 10. CANopen 实际开发关键点

### 10.1 Node-ID 分配

| Node-ID | 典型角色 |
|---------|---------|
| 0x00 | 广播地址 |
| 0x01 | NMT 主站 |
| 0x02 – 0x7F | 从站 (127 个最大) |

### 10.2 通信周期规划

```
周期时间 = SYNC 周期 (0x1006)

推荐:
  伺服定位: 1ms – 4ms
  电机速度控制: 2ms – 10ms
  温度/状态监测: 50ms – 200ms
  SDO 配置: 无实时性要求
```

### 10.3 PDO 禁止时间 (Inhibit Time)

如果使用异步 TPDO (传输类型 0xFF)，强烈建议设置禁止时间 (0x1800.03)，防止高频率状态变化导致总线被单节点占满。

```
写 0x1800.03 = 100 (单位 100μs = 10ms)
→ 同一 TPDO 两次发送间隔至少 10ms
```

### 10.4 对象字典编辑器

开发阶段推荐使用对象字典编辑工具生成 EDS (Electronic Data Sheet) 文件：
- **CANeds** (Vector)
- **CANopen DeviceDesigner** (port GmbH)
- 手写 EDS (ASCII 格式, 可参考 CiA 306 规范)
